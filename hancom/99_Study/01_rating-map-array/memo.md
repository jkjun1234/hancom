# 집 학습 메모 — Rating 컴포넌트 (map + 배열 + 삼항)

> 260710 강의장 실습 중 남긴 메모. 집에서 `coaching-prompt.md`로 드릴 돌릴 때 참고용.
> 대상 코드: `hancom/05_react/my-app/src/components/25/Rating.jsx`

## 오늘 상황
- 강의장에서 Alert(객체 map 실습)까지는 이해하고 고침. `borderColor`(camelCase), `style={{ }}`는 객체 하나라는 것까지 소화함.
- **Rating은 낯선 문법이 겹쳐서 집에서 다시 보기로 함.**

## 집에서 집중할 포인트 3개

```jsx
{[...Array(5)].map((_, i) => (
    <span key={i}>{i < score ? '⭐' : '☆'}</span>
))}
```

1. **`[...Array(5)]`** — 왜 그냥 `Array(5)`가 아니라 스프레드로 펼치는가?
   - `Array(5)`는 length만 5인 "빈 칸(empty) 배열" → `.map`이 안 돌아감.
   - `[...Array(5)]` 로 펼치면 `[undefined, undefined, ...]` 진짜 5칸 배열이 되어 `.map` 가능.
   - 직접 확인 드릴: 콘솔에 `Array(5)` vs `[...Array(5)]` 찍어 차이 보기.

2. **`.map((_, i) => ...)`**
   - 첫 인자 `_` = 값(안 씀, 버림 관례). `i` = 인덱스 0~4.
   - `key={i}` 가 왜 필요한지 (React 리스트 렌더 시 각 요소 식별).

3. **`i < score ? '⭐' : '☆'`**
   - 삼항 연산자로 별 채우기. `score=3`이면 i=0,1,2만 꽉 찬 별.

## 오늘 배운 것과의 연결 (재사용 패턴)
- **"배열/객체 → 화면으로 변환"** = Alert의 객체 map, Rating의 배열 map 둘 다 같은 뿌리.
- **삼항 연산자** = 조건부 렌더링. 앞으로 미니 프로젝트에서 "상태에 따라 다른 UI" 낼 때 계속 씀.

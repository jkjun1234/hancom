# 📋 예상 채점 루브릭 (예측)

> **목적:** 이전 회차(HTML+CSS) AI 채점 루브릭을 근거로, 이번 HTML+CSS+**JS** 프로젝트의 채점 기준을 예측한다.
> **주의:** 실제 기준은 강사님이 배포한다. 이 문서는 "만점 구조로 설계됐는지" 자가 점검용 예측이다.
> **AI 채점:** 40점 만점 | **동료 투표:** 별점 × 3점 | **최종:** 최대 97점 (이전 회차와 동일 가정)

---

## 예측의 전제

1. 강사님도 AI로 루브릭을 작성하는 것으로 보이므로, **이전과 유사한 표 형식·배점 스타일**일 가능성이 높다.
2. 이전과 달라진 점 3가지를 반영해 예측한다:
   - **주제 자유화** → "참고 사이트 분석"이 필수에서 빠지고 "기획서 충실도"가 새 항목이 될 수 있음
   - **기획서(md+pdf) 필수 제출** → 기획 관련 배점 신설 가능성
   - **JavaScript 추가** → JS 동작·완성도 항목 신설 확실시
3. 그래서 아래에 **시나리오 A / B 두 가지**를 모두 준비하고, 각 항목마다 "우리 대응"으로 만점 근거를 적었다.

---

## 시나리오 A — 이전 구조 유지형 (참고 사이트 분석 존치)

이전 6개 항목(R1~R6)을 유지하면서 JS 항목만 끼워 넣는 경우.

| 항목 | 예상 배점 | 세부 기준 | 우리 대응 |
|------|:--------:|-----------|-----------|
| A1. 참고 사이트 발굴 | 2 | 사이트 이름 + URL 명시 | 참고 사이트 분석 섹션(Flexbox Froggy 등) URL 포함 |
| A2. 참고 사이트 분석 | 4 | 레이아웃·색·UX 구체 근거 + 주관 의견 | 각 사이트 "무엇을/좋은 점/우리 반영" 서술 |
| A3. 개념 정리 | 6 | HTML·CSS·JS 정리 포함, 간결·명료 | 개념 정리 노트(접기) 3영역 전부 |
| A4. 퀴즈 품질 | 8 | 문항 수·관련성·정답·해설 | 커버리지 맵 전수 문항 + 해설 패널(이유+코드+개념태그) |
| A5. 시맨틱 HTML | 5 | 문서 구조·시맨틱 태그·유효 중첩 | header/main/section/article/nav/footer |
| A6. CSS 품질 | 5 | 외부 CSS·디자인·선택자 관리 | 외부 1파일, `:root` 변수, kebab-case, 반응형 |
| **A7. JS 완성도(신설)** | 6 | 동작 오류 없음·DOM/이벤트 활용·가독성 | 채점·랭킹·화면전환 무오류, 배운 문법 총동원 |
| A8. 완성도·배포 | 4 | 가독성·깨진 요소 없음·Netlify 접속 | 시크릿 창 접속 확인, 반응형 |
| **합계** | **40** | | |

## 시나리오 B — 주제 자유형 (기획서 충실도 신설) 〔가능성 더 높음〕

주제가 자유이고 기획서가 필수이므로, "참고 사이트 분석"이 빠지고 "기획·주제"와 "JS"가 강화되는 경우.

| 항목 | 예상 배점 | 세부 기준 | 우리 대응 |
|------|:--------:|-----------|-----------|
| B1. 주제·기획 명확성 | 6 | 기획서 md+pdf 제출, 목표·컨셉 명확 | 기획서 2종 제출, 스토리·페르소나·기능 명세 완비 |
| B2. 학습 정리 품질 | 6 | HTML·CSS·JS 핵심 정리 | 개념 정리 노트(접기) + 커버리지 맵 |
| B3. 퀴즈 품질 | 8 | 유형 다양·정답·해설·관련성 | 3유형(4지선다·코드·OX) + 해설 패널 |
| B4. 시맨틱 HTML | 5 | 문서 구조·시맨틱 태그·유효 중첩 | header/main/section/article/nav/footer |
| B5. CSS 품질 | 5 | 외부 CSS·디자인·선택자 관리 | `:root` 변수, kebab-case, 반응형, 애니메이션 |
| **B6. JS 완성도** | 8 | 동작 오류·DOM/이벤트/배열/저장소 활용·인터랙션 | 실시간 반응, 채점, localStorage 랭킹, 문항 배열(questions.js) 렌더링 — 100% 학습 문법 |
| B7. 완성도·배포 | 2 | Netlify 정상 접속·깨진 요소 없음 | 시크릿 창 확인 |
| **합계** | **40** | | |

## 감점·주의 (엣지케이스 예측)

| 상황 | 예상 처리 |
|------|-----------|
| Netlify 링크 404 / 접속 불가 | 해당 항목 0점 (이전: URL 404 = 0점) → **제출 전 시크릿 창 접속 필수** |
| 퀴즈에 정답·해설 없음 | 퀴즈 품질 -2~3점 → 전 문항 `why`·`answer` 필수 |
| 콘솔 에러로 기능 정지 | JS 완성도 대폭 감점 → 자가 테스트에서 콘솔 에러 0 확인 |
| 인라인 style 남용 | CSS 품질 감점 → 외부 CSS 원칙 |
| 시맨틱 태그 전무 | 시맨틱 0~0.5점 → 구조 태그 사용 |
| md/pdf 내용 불일치 | 기획 항목 감점 → PDF는 md에서 자동 변환 |
| AI 복붙 의심(주제 무관) | 이전엔 R1+R2 0점. 본 프로젝트는 실습 출처(`source`) 명시로 학습 연계 입증 |

## 예측 근거

- **이전 루브릭(R1~R6, 40점)** 을 뼈대로 삼음. 배점 총합 40점, 세부 항목표, 감점 기준, 별점 환산 구조를 그대로 계승.
- **달라진 점 반영:** 주제 자유 → 참고 사이트 분석의 필수성 약화(시나리오 B), 기획서 필수 → 기획 항목 신설(B1), JS 추가 → JS 완성도 신설(A7/B6, 이전엔 없던 항목).
- **두 시나리오 공통 결론:** 어느 쪽이 나와도 이 기획은 각 항목에 대응 기능이 있어 **만점 구조**다. 특히 JS 완성도(가장 불확실하고 배점 큰 신설 항목)를 실시간 반응·채점·랭킹·localStorage로 두껍게 커버하며, 배운 문법만으로 구현해 "동작 오류 없음" 항목의 리스크도 최소화한다.
- **투표(60%) 대비:** 같은 반 타깃에 맞춘 "부담 없는 복습 + 즉각 피드백 + 랭킹 경쟁"으로 재방문·재도전을 유도.

# CSS 리셋: * 선택자로 기본 여백 0부터 시작하기

> **한 줄 요약**: 브라우저가 기본으로 넣는 여백 때문에 화면 가장자리에 틈이 생기는데, `*` 선택자로 `margin`·`padding`을 0으로 초기화하고 시작하면 레이아웃을 정확히 제어할 수 있다.

새 HTML을 열면 아무것도 안 했는데 화면 가장자리에 흰 여백이 있다. 이건 브라우저가 넣는 **기본 스타일** 때문이다. 작업 전에 이걸 0으로 밀고 시작하는 게 **CSS 리셋**이다.

![리셋 전 — 화면 왼쪽 위·가장자리에 흰 여백(틈)이 있는 모습](C:/Users/jun/Desktop/블로그 업로드용 이미지/06_css-reset-01-before.png)

## HTML

```html
<body>
    <h1>CSS 리셋</h1>
    <p>main.css 맨 위 * 리셋 3줄을 지웠다 넣었다 하며 화면 왼쪽 위 흰 틀을 비교해 보자</p>
</body>
```

## CSS

```css
/*
    여백 및 크기 초기화
    이렇게 CSS를 초기화해야 작업 시 CSS 제어를 편하게 관리할 수 있다.
*/
* {
    margin: 0;             /* 기본 바깥 여백 제거 */
    padding: 0;            /* 기본 안쪽 여백 제거 */
    box-sizing: border-box; /* 크기 계산 방식 변경 */
}

h1 {
    color: green;
}

p {
    color: peru;
}
```

핵심:

- **`*` (전체 선택자)** → 페이지의 **모든 요소**를 한 번에 선택한다. 여기에 리셋 규칙을 주면 전부 적용된다.
- **`margin: 0` / `padding: 0`** → 브라우저 기본 여백을 없앤다. `<body>`, `<h1>`, `<p>` 등에 은근히 붙어 있던 틈이 사라진다.
- **`box-sizing: border-box`** → 크기 계산 방식을 바꾼다. (아래에서 설명)

## box-sizing: border-box 가 왜 중요할까

기본값(`content-box`)에서는 `width`가 **내용만**의 너비다. 그래서 padding·border를 주면 요소가 그만큼 **더 커진다.**

- `content-box`(기본) : `width: 100px` + `padding: 20px` → 실제 폭 **140px** (예상과 어긋남)
- `border-box`(리셋 후) : `width: 100px` + `padding: 20px` → 실제 폭 **100px** (padding·border가 안쪽에 포함)

즉 `border-box`로 두면 **내가 정한 `width`가 곧 실제 크기**라서 레이아웃 계산이 훨씬 직관적이다.

## 짚고 넘어갈 점

- 리셋 3줄을 넣고 빼며 비교하면, 화면 가장자리의 흰 틈이 생겼다 사라지는 걸 눈으로 확인할 수 있다.
- `*` 리셋은 가장 간단한 방식이고, 실무에선 더 정교한 리셋(예: Normalize.css)을 쓰기도 한다. 하지만 학습 단계에선 이 3줄로 충분하다.
- 리셋은 항상 CSS **맨 위**에 두어야 다른 규칙보다 먼저 깔린다.

## 오늘의 한 줄 정리

작업 시작 전 `* { margin:0; padding:0; box-sizing:border-box }` 3줄로 브라우저 기본값을 밀어두면, 여백·크기가 예상대로 동작해 레이아웃 잡기가 편해진다.

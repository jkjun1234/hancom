## 0. 오늘의 요약

- **권한 모드**: `claude --dangerously-skip-permissions`로 켜는 bypassPermissions는 파일 수정·명령 실행을 전부 자동 승인하는 모드다. 편하지만 위험해서 도커·VM 같은 격리 환경 전용으로 권장된다.
- **`/statusline`**: 클로드 코드 하단에 컨텍스트 사용률, 세션 비용, git 브랜치, 모델명 같은 걸 항상 보이게 하는 설정. 자연어로 원하는 구성을 말하면 알아서 스크립트를 만들어준다.

- 이 글의 이미지는 대부분 자리표시로 남겨뒀다 — 아래에서 설명하는 이유 참고.

> ⚠️ 이 글 이미지 중 `skills.ag`만 실제로 접속해 캡처한 화면이다. bypassPermissions·statusline은 터미널/세션 상호작용이라 자동 캡처가 애매해서 자리만 표시해뒀다. 직접 실행한 화면으로 채워 넣으면 된다.

## 1. 권한 모드 — `bypassPermissions`가 뭘 건너뛰는가

Claude Code는 파일을 고치거나 명령어를 실행하기 전에 원래 한 번씩 물어본다. `bypassPermissions`는 그 확인 절차를 전부 꺼버리는 모드다.

```bash
claude --dangerously-skip-permissions
```

- 이 옵션을 켜면 **파일 수정, 명령 실행을 모두 자동으로 승인**한다. 매번 "이거 해도 돼?"라고 묻지 않는다.
- 대신 이름에 이미 경고가 박혀 있다(`dangerously`). 잘못된 명령이 그대로 실행될 수 있기 때문에, **도커나 VM처럼 실제 시스템과 격리된 환경에서 쓰는 게 권장된다.** 지금 이 실습 PC처럼 내 실제 파일 시스템에 바로 접근하는 환경에서 이 모드를 상시로 켜두는 건 추천되지 않는다.
- 세션 도중에는 `Shift+Tab`으로 권한 모드를 순환 전환할 수 있다. 즉 처음부터 이 옵션으로 켜지 않아도, 작업하다가 필요할 때만 모드를 바꿔 쓸 수 있다는 뜻이다.

🖼️ **이미지 자리** — 터미널에서 `claude --dangerously-skip-permissions`로 세션을 켠 직후 화면, 또는 `Shift+Tab`으로 권한 모드가 순환되는 화면(모드 이름이 하단에 바뀌어 표시되는 부분)

### 왜 이게 중요했나

지금까지는 매 작업마다 승인 프롬프트가 뜨는 걸 당연하게 여겼는데, 그게 **기본 안전장치**였다는 걸 오늘 처음 인식했다. 이 모드를 끄고 켜는 걸 배운 것보다, "왜 격리 환경에서만 권장되는가"를 이해한 게 더 남는 부분이었다.

## 2. `/statusline` — 클로드 코드 하단 정보창 만들기

`/statusline`은 Claude Code 입력창 바로 아래에 **항상 떠 있는 상태 표시줄**을 커스터마이즈하는 기능이다.

- 기본적으로 컨텍스트 사용률, 세션 비용, 현재 git 브랜치, 모델명 같은 정보를 표시하도록 설정할 수 있다.
- 방식이 재밌었다 — 내가 "이런 걸 보여줘"라고 자연어로 말하면, Claude가 그에 맞는 **스크립트를 직접 생성하고 설정 파일을 수정**해준다. 코드를 내가 짜는 게 아니라 원하는 결과를 말로 설명하는 쪽에 가깝다.
- 적용하려면 셸 명령 실행에 대한 워크스페이스 신뢰 승인이 한 번 필요하다.

오늘은 미리 참고 자료(설정 예시 파일)를 받아둔 게 있어서, 다음에 그 파일을 클로드에 보여주고 "이대로 적용해줘"라고 요청하면 된다.

🖼️ **이미지 자리** — `/statusline` 적용 전/후 비교(적용 전: 하단에 아무 정보도 없는 기본 입력창 / 적용 후: 컨텍스트 사용률·브랜치명 등이 표시된 상태줄), 2장

## 3. 스킬 생태계 — 오늘 알게 된 것 넷

Claude Code는 "스킬"이라는 형태로 특정 작업 습관이나 절차를 통째로 설치해서 쓸 수 있다. 오늘 이 네 가지를 알게 됐다.

### 3-1. `grill-me` — 계획을 끝까지 캐묻는 스킬

작업을 시작하기 전에 방향을 제대로 잡았는지, 애매한 부분은 없는지 **AI가 먼저 질문을 던져서 확인**해주는 스킬이다. 결정 트리를 따라 의존성(이걸 정하려면 저것부터 정해야 하는 것들)을 하나씩 풀어나가고, 질문마다 추천 답변도 같이 제시해준다.

지금 이 CLAUDE.md에 있는 "코딩 전에 생각하기" 원칙 — 가정을 넘겨짚지 말고, 선택지가 여러 개면 물어보라는 것 — 을 스킬로 만들어놓은 셈이라, 왜 필요한지 바로 이해가 됐다.

🖼️ **이미지 자리** — `grill-me` 스킬을 켜고 질문을 받는 터미널 화면

### 3-2. `caveman` — 토큰을 아끼는 스킬

이름 그대로 "원시인처럼 짧게 답하게" 만드는 모드다. GitHub 저장소(`https://github.com/JuliusBrussee/caveman`)를 클로드에게 알려주고 적용을 요청하면 설치된다.

- `/caveman`, `/caveman lite`처럼 강도가 다른 모드가 여러 개 있고, 커밋 메시지·코드리뷰 전용(`caveman-commit`, `caveman-review`) 같은 세부 기능도 있다.
- 토큰을 약 65% 줄여준다는 수치와 함께, 오히려 정확도가 올라갔다는(+26%p) 연구 결과도 같이 소개돼 있었다. **짧게 답하는 게 단순히 비용 절감이 아니라, 불필요한 군더더기를 줄여서 정확도에도 도움이 될 수 있다**는 점이 의외였다.

### 3-3. `/project-plan` — 내가 세팅한 스킬

이건 남이 만든 걸 가져온 게 아니라 **직접 구현해본 나만의 스킬**이다. 프로젝트 아이디어를 기능 목록과 구현 순서로 정리해주는 용도로 만들었다. 예를 들어 "이런 앱 만들고 싶어"라고 말하면, 우선순위가 매겨진 기능 목록과 어떤 순서로 만들지를 정리해준다.

다른 두 스킬(`grill-me`, `caveman`)이 "설치해서 쓰는" 스킬이라면, 이건 "내 작업 방식에 맞춰 직접 만들어본" 스킬이라는 점에서 결이 달랐다.

### 3-4. `skills.ag` — 스킬을 사고파는 장터

마지막으로 들른 곳은 스킬 자체를 상품처럼 사고파는 사이트였다.

<img src="images/blog_02_skillsag_01.png" width="800">

카테고리가 코딩/에이전트, 자동화, 디자인, 마케팅, 데이터/엑셀처럼 세분화돼 있고, 배너에 걸린 "유튜브 자동화 공장 만들기"처럼 여러 스킬을 하나의 파이프라인으로 묶어 파는 상품도 있었다. **스킬 하나하나가 "이 작업을 이렇게 처리해라"는 노하우인데, 그게 그대로 상품이 될 수 있다**는 게 신기했다. 노트에 적어둔 감상 그대로 옮기면 — "나도 한번 팔아볼까?" 싶은 생각이 들 만했다.

## 4. 마무리 — 언제 뭘 켤지

오늘 배운 걸 실제로 언제 쓸지 정리하면:

- **`bypassPermissions`**: 반복적인 승인 프롬프트가 진짜 방해될 때, 단 반드시 격리된 환경(도커/VM)에서만. 지금 이 실습 PC에서 상시로 켜두진 않는다.
- **`/statusline`**: 세션 비용이나 컨텍스트 사용량을 자주 확인하고 싶을 때 한 번 설정해두면 계속 유용하다.
- **`grill-me`**: 새 프로젝트를 시작하기 전, 방향이 애매하다 싶을 때.
- **`caveman`**: 단순 반복 작업이라 긴 설명이 필요 없을 때.
- **`/project-plan`**: 아이디어는 있는데 뭐부터 만들지 막막할 때.

다음엔 오늘 만든 `/project-plan` 스킬을 실제 미니 프로젝트에 한번 써보고 싶다.

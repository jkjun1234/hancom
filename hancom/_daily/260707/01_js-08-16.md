## 08. 이벤트 — 버튼을 누른 횟수 세기

가장 기본이 되는 클릭 이벤트. 버튼을 누를 때마다 숫자가 하나씩 올라가게 만들었다.

```html
<button id="btn">여기를 클릭!</button>
<p id="count">0번 눌렀어요</p>
```

```js
let n = 0;  // 누른 횟수를 담을 변수
const btn = document.querySelector("#btn");
const out = document.querySelector("#count");

btn.addEventListener("click", () => {
  n++;                               // 누를 때마다 1 증가
  out.textContent = `${n}번 눌렀습니다`;
});
```

## 09. 이미지 바꾸기 — src를 JS로 제어하기

이번엔 HTML에서 `src`를 안 정하고, JS에서 이미지 주소를 넣고 바꿨다. 그림을 클릭할 때마다 두 사진이 번갈아 나온다.

```js
const IMG_A = "https://picsum.photos/100?random=1";
const IMG_B = "https://picsum.photos/100?random=2";

const myImage = document.querySelector("#pic");
myImage.setAttribute("src", IMG_A);   // 처음 그림 넣기

myImage.onclick = () => {
  const now = myImage.getAttribute("src");   // 지금 걸린 주소 읽기
  myImage.setAttribute("src", now === IMG_A ? IMG_B : IMG_A);
};
```

|:---:|:---:|

여기서 두 가지가 새로웠다. 하나는 태그 속성을 **읽을 땐 `getAttribute`, 바꿀 땐 `setAttribute`** 를 쓴다는 거. 다른 하나는 이벤트를 `addEventListener` 대신 **`.onclick = () => {}`** 로도 걸 수 있다는 거였다. 둘 다 클릭에 반응하는 건 똑같은데, `onclick`은 하나만 연결되고 뒤에 또 넣으면 덮어써진다는 차이가 있다고 한다. (그림은 picsum이라는 더미 이미지 서비스라 인터넷이 있어야 뜬다.)

## 10. localStorage — 새로고침해도 기억하기

입력한 이름을 브라우저에 저장해두고, 다음에 열어도 그 이름으로 인사하게 만들었다. 이게 은근 신기했다. 변수는 새로고침하면 다 날아가는데, localStorage에 넣으면 남아 있다.

```js
const greet = document.querySelector("#greet");
const input = document.querySelector("#name");

// 열 때 저장된 이름이 있으면 꺼내서 인사
const saved = localStorage.getItem("name");
if (saved) greet.textContent = `안녕, ${saved}`;

document.querySelector("#save").addEventListener("click", () => {
  const myName = input.value;
  if (!myName) return;                     // 비어 있으면 그냥 끝
  localStorage.setItem("name", myName);    // 저장
  greet.textContent = `안녕, ${myName}`;
});

document.querySelector("#reset").addEventListener("click", () => {
  localStorage.removeItem("name");         // 삭제
  greet.textContent = "안녕하세요";
});
```

|:---:|:---:|
| <img src="C:/Users/jun/Desktop/블로그 업로드용 이미지/js10_before.png" width="280" alt="저장 전"> | <img src="C:/Users/jun/Desktop/블로그 업로드용 이미지/js10_after.png" width="280" alt="저장 후"> |

`setItem(키, 값)`으로 넣고 `getItem(키)`로 꺼낸다. 키-값으로 저장된다는 게 포인트. `if (!myName) return;` 은 입력칸이 비었을 때 아무것도 안 하고 함수를 빠져나가는 방어 코드인데, 이런 거 하나 넣어두니 빈 값이 저장되는 실수를 막을 수 있었다.

## 11. 반복문 — 목록을 코드로 찍어내기

입력한 숫자만큼 `<li>`를 만들어 목록에 채우는 실습. `for`랑 `while` 둘 다 써봤다. for는 1부터 위로, while은 입력값부터 1까지 거꾸로 내려오게 했다.

```js
const n = document.querySelector("#n");
const out = document.querySelector("#out");

// 반복 버튼 — for로 1 → count
document.querySelector("#run").addEventListener("click", () => {
  out.innerHTML = "";                 // 기존 목록 비우기
  const count = Number(n.value);
  for (let i = 1; i <= count; i++) {
    const li = document.createElement("li");   // li 만들고
    li.textContent = `${i} 번째 사과~`;
    out.appendChild(li);                        // ul 안에 넣기
  }
});

// 카운트다운 버튼 — while로 count → 1
document.querySelector("#down").addEventListener("click", () => {
  out.innerHTML = "";
  let i = Number(n.value);
  while (i > 0) {
    const li = document.createElement("li");
    li.textContent = i;
    out.appendChild(li);
    i--;
  }
});
```

| 반복 (for, 1→4) | 카운트다운 (while, 4→1) |
|:---:|:---:|
| <img src="C:/Users/jun/Desktop/블로그 업로드용 이미지/js11_for.png" width="280" alt="for 결과"> | <img src="C:/Users/jun/Desktop/블로그 업로드용 이미지/js11_while.png" width="280" alt="while 결과"> |

여기서 세 개를 새로 알았다.
- `out.innerHTML = ""` — 목록을 통째로 비우는 방법. 버튼 다시 누를 때 이전 결과가 안 쌓이게.
- `document.createElement("li")` — 태그를 JS로 새로 만들기.
- `.appendChild(li)` — 만든 걸 부모(`<ul>`) 안에 자식으로 넣기.

만들고(create) → 넣는다(append) 이 두 단계가 한 세트라는 감이 왔다.

## 12. 문자열 메서드 — 글자를 다루는 도구들

입력한 글자를 이런저런 문자열 메서드로 바꿔서 보여줬다.

```js
const s = document.querySelector("#s");
const out = document.querySelector("#out");

document.querySelector("#go").addEventListener("click", () => {
  const text = s.value;
  out.innerHTML =
    `글자 수 (length): ${text.length}` + "<br>" +
    `대문자 (toUpperCase): ${text.toUpperCase()}` + "<br>" +
    `e → E (replace): ${text.replace("e", "E")}`;
});
```

## 13. 배열 메서드 — join과 push

과일 배열을 만들고, 입력한 과일을 배열에 추가해서 화면에 뿌렸다.

```js
let fruits = ["사과", "바나나"];
const fruit = document.querySelector("#fruit");
const out = document.querySelector("#out");
const info = document.querySelector("#info");

const render = () => {
  out.textContent = fruits.join(" ");            // 배열을 문자열로 이어서 보여주기
  info.textContent = `개수(length): ${fruits.length}`;
};
render();

document.querySelector("#add").addEventListener("click", () => {
  if (!fruit.value) return;
  fruits.push(fruit.value);   // 배열 끝에 실제로 값 추가
  fruit.value = "";
  render();
});
```

| 추가 전 | "포도" 추가 후 |
|:---:|:---:|
| <img src="C:/Users/jun/Desktop/블로그 업로드용 이미지/js13_before.png" width="240" alt="추가 전"> | <img src="C:/Users/jun/Desktop/블로그 업로드용 이미지/js13_after.png" width="240" alt="추가 후"> |

여기서 `join`이랑 `push`가 뭐가 다른지 계속 헷갈렸는데, 정리하니 이랬다.

- **`push`** 는 배열에 값을 **진짜로 추가**한다. `fruits`가 실제로 3개가 된다. (원본이 바뀜)
- **`join`** 은 배열을 건드리지 않고, 항목들 **사이에 구분자만 끼워서 문자열 하나로 이어 보여줄** 뿐이다. `["사과","바나나"].join(" ")` → `"사과 바나나"`. 배열 개수는 그대로 2개.

즉 push는 "데이터를 바꾸는 것", join은 "보여주기 위해 문자열로 만드는 것"이라고 이해했다. 역할이 아예 다른 거였다.

## 14. 객체 — 값을 이름표로 묶기

관련된 값을 `key: value`로 묶은 객체를 만들고, 점(`.`)으로 값을 꺼내고 바꿔봤다.

```js
let person = { name: "콩이", age: 10 };
const out = document.querySelector("#out");

const render = () => {
  out.textContent = `${person.name} (${person.age}살)`;
};
render();

// 나이 +1
document.querySelector("#up").addEventListener("click", () => {
  person.age++;
  render();
});

// 이름 토글 (깨 ↔ 두부)
document.querySelector("#rename").addEventListener("click", () => {
  person.name = person.name === "깨" ? "두부" : "깨";
  render();
});
```

| <img src="C:/Users/jun/Desktop/블로그 업로드용 이미지/js14_before.png" width="180" alt="처음 콩이 10살"> | <img src="C:/Users/jun/Desktop/블로그 업로드용 이미지/js14_after.png" width="180" alt="콩이 12살"> | <img src="C:/Users/jun/Desktop/블로그 업로드용 이미지/js14_rename.png" width="180" alt="이름이 깨로 바뀜"> |

`person.age++`처럼 점으로 객체 안의 값에 바로 접근해서 바꿀 수 있는 게 편했다. 이름도 똑같이 `person.name`에 새 값을 넣으면 바뀐다. **이름 바꾸기** 버튼은 누를 때마다 `깨` ↔ `두부`로 번갈아 바뀌게 해뒀는데, 처음 값이 "콩이"라서 첫 클릭 때는 "콩이 → 깨"로 넘어가고 그다음부터 깨/두부를 오간다.

솔직히 고백하면, **색상 바꾸기 버튼은 아직 못 만들었다.** `out.setAttribute("color", #blue)` 라고 썼다가 `#blue`가 문법 오류라 스크립트 전체가 안 돌아가는 걸 뒤늦게 알았다. 색을 바꾸는 건 `out.style.color = "blue"` 쪽이 맞는 것 같아서, 이건 다음에 제대로 고쳐볼 생각이다. (그래서 위 코드에선 그 버튼은 빼고 정리했다.)

## 15. 클래스 — 같은 틀로 여러 개 찍어내기

`Dog`라는 틀(클래스)을 만들고, 그 틀로 강아지 두 마리를 찍어냈다.

```js
class Dog {
  constructor(name) {   // new로 만들 때 자동 실행, 받은 이름을 보관
    this.name = name;
  }
  bark() {              // 메서드(클래스 안의 함수)
    return `${this.name}: 왈왈!`;
  }
}

const poppy = new Dog("뽀삐");
const choco = new Dog("초코");

document.querySelector("#bark").addEventListener("click", () => {
  out.textContent = `${poppy.bark()} ${choco.bark()}`;
});
```

## 16. 디버깅 — 콘솔로 값 들여다보기

마지막은 코드가 뜻대로 안 될 때 값을 확인하는 방법. `console.log`로 값을 찍어 F12 → Console 탭에서 봤다.

```js
const price = 1000;
const count = 3;
console.log("price:", price);        // price: 1000
console.log("총합:", price * count); // 총합: 3000

console.log(typeof price);  // "number"
console.log(typeof "3");    // "string" — 따옴표가 있으면 글자다!

// 일부러 오타를 내봤다: 진짜 id는 #buy인데 #by라고 씀
const btn = document.querySelector("#by");  // 못 찾아서 null
btn.addEventListener("click", () => {});    // null에 붙이려다 에러
```

실제로 콘솔에 찍힌 결과:

```text
price: 1000
총합: 3000
number
string
❌ TypeError: Cannot read properties of null (reading 'addEventListener')
```

`console.log`는 JS의 프린트 같은 거라, 중간중간 값을 찍어보면서 "여기까진 값이 제대로 들어오네" 하고 확인할 수 있었다. 그리고 일부러 id를 틀리게(`#by`) 써봤더니 `Cannot read properties of null` 에러가 났다. `querySelector`가 요소를 못 찾으면 `null`을 주는데, 그 `null`에 `addEventListener`를 붙이려다 터진 거다. 앞으로 이 에러를 만나면 **id 철자부터 확인**하면 되겠구나 싶었다. (에러 메시지가 무섭게 생겼어도 사실 "그거 없어요"라는 뜻이었다.)

`typeof "3"`이 `"string"`인 것도 다시 확인했다. 따옴표만 있으면 숫자처럼 보여도 글자다.

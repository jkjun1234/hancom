# React 프로젝트 파일 구조 파악 (05_react/my-app)

강의장 실습 시간에 `05_react/my-app` 전체 파일 구조를 훑어보고, `App.jsx`를 직접 만져보면서 각 파일이 뭘 하는지 파악한 세션.

## 정리된 이해

- **`index.html`** — `<div id="root"></div>` 딱 하나, 빈 껍데기. 실제 내용은 없음.
- **`main.jsx`** — `createRoot(document.getElementById('root')).render(<App />)`로, `index.html`의 `#root`를 찾아서 그 자리에 `App` 컴포넌트를 그려 넣는 진입점(entry point).
- **`App.jsx`** — 실제로 화면에 보이는 내용(JSX)을 담당. `localhost:5173`에서 보이는 모든 요소가 여기서 나옴.
- **`index.css`** — `main.jsx`가 import. `body`, `#root`, `h1`, `:root` 변수처럼 **태그/전역 단위**를 대상으로 하는 사이트 전체의 기본값(색상, 폰트, 다크모드 등).
- **`App.css`** — `App.jsx`가 import. `.hero`, `.ticks`처럼 **App 컴포넌트 안의 특정 요소 전용** 스타일.

처음엔 `index.css`/`App.css` 차이를 "페이지가 다르다"는 기준으로 생각했는데, 지금 프로젝트엔 페이지가 하나뿐이라 그 구분은 성립하지 않았음. 실제 기준은 **"누가 import하냐"** + **"전역 기본값이냐 vs 특정 클래스 전용이냐"**.

## 재사용 패턴 메모

- 새 프레임워크/라이브러리의 구조가 안 잡힐 때는 **진입점부터 역추적**하면 빠르다: `index.html`의 빈 껍데기 → `main.jsx`(연결) → `App.jsx`(내용). Vue든 다른 프레임워크든 "빈 div + JS가 그 안을 채운다"는 패턴은 거의 동일.
- CSS가 여러 파일로 나뉘어 있으면 **"누가 import하는지" + "전역 대상 vs 특정 클래스 대상"** 두 가지로 역할을 구분한다. 컴포넌트가 늘어나도 이 기준은 그대로 적용됨 (전역 1개 + 컴포넌트별 각자).

## 이전 개념과의 연결

HTML/CSS/JS를 파일로 나눠 쓰던 것과 원리는 같음 — "구조·표시 내용(App.jsx) / 디자인(css) / 동작"을 분리해서 관리하는 것. React에서는 여기에 "컴포넌트 단위로 또 나눈다"는 개념이 하나 더 얹힘 (→ 다음 응용에서 확인, [02_home-이어하기.md](02_home-이어하기.md) 참고).

## 강의장에서 있었던 일

`05_react/my-app` 파일 구조(`index.html` / `main.jsx` / `App.jsx` / `index.css` vs `App.css`)를 훑어보고 직접 만져보면서 역할을 파악함. 처음엔 `index.css` 역할을 "다른 페이지용"으로 오해했다가, "누가 import하냐 + 전역 대상이냐 특정 클래스 전용이냐" 기준으로 정리함. 자세한 내용은 [01_react-file-structure.md](01_react-file-structure.md).

## 남겨둔 응용 문제

지금 `App.jsx` 안에 `count`/`setCount`를 쓰는 카운터 버튼 부분이 있음. 이걸 **별도의 컴포넌트 파일로 분리**해보기 (오늘 배운 "역할 분리" 개념을 그대로, 하지만 App.jsx 안에서 텍스트 바꾸는 것과는 다른 상황에 적용).

진행 순서 힌트 (정답 코드 아님, 스스로 채워볼 것):

1. `src/Counter.jsx` 새 파일 생성. 카운터 버튼 관련 JSX + `useState` 로직을 그쪽으로 옮기기.
2. `src/Counter.css` 새 파일 생성. `App.css`에 있던 `.counter` 관련 스타일을 옮기고, `Counter.jsx`에서 import.
3. `App.jsx`에서는 그 자리에 `<Counter />` 로 불러오기 (import 잊지 말 것).
4. 확인할 것:
   - 화면에 이전과 똑같이 나오는가?
   - `index.css`의 전역 스타일(색상 변수, 폰트 등)은 `Counter.jsx`에서도 그대로 적용되는가? → 왜 그런지 설명할 수 있어야 함 (전역이라는 뜻이 뭔지).
   - `App.css`에 남겨둔 `.hero` 스타일은 문제없이 그대로 동작하는가?
5. 막히면: "main.jsx → App.jsx" 연결했던 방식(`import` + JSX 태그로 부르기)이 "App.jsx → Counter.jsx"에도 **똑같이** 적용된다는 걸 힌트로.

풀고 나면 재사용 메모 한 줄: "컴포넌트를 나눌 때 무엇을 기준으로 파일을 쪼갰는지" 남기기.

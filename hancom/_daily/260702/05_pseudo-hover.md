# 가상 클래스로 링크 상태 꾸미기 (:link :visited :hover :active)

> **한 줄 요약**: 링크는 방문 전·방문 후·마우스 올림·클릭 중이라는 4가지 상태를 가지며, 가상 클래스로 각 상태의 색을 따로 지정할 수 있다.

`<a>` 링크는 사용자의 행동에 따라 상태가 바뀐다. **가상 클래스(pseudo-class)** `:`을 쓰면 "이 상태일 때만" 스타일을 줄 수 있다.

## HTML

```html
<body>
    <h1>가상 클래스: hover</h1>
    <a href="https://example.test" target="_blank">마우스를 올려보세요.</a>
</body>
```

## CSS

```css
/* 링크 클릭 전(방문 전) */
a:link {
    color: red;
}

/* 링크 방문 후 */
a:visited {
    color: aqua;
}

/* 링크에 마우스 올렸을 때 */
a:hover {
    color: green;
}

/* 링크 활성화(누르는 중)일 때 */
a:active {
    color: rgb(255, 238, 0);
}
```

네 가지 상태:

- **`:link`** → 아직 방문하지 않은 링크 (기본 상태)
- **`:visited`** → 한 번이라도 방문(클릭)한 적 있는 링크
- **`:hover`** → 마우스를 올린 순간
- **`:active`** → 마우스로 누르고 있는 순간(클릭 중)

실제로 링크에 마우스를 올려보면 `a:hover` 규칙이 적용되어 텍스트가 초록색으로 바뀐다.

같은 링크를 방문한 기록이 남은 뒤 다시 보면 `a:visited` 규칙이 적용되어 하늘색(aqua)으로 표시된다.

## 짚고 넘어갈 점 — :visited가 바로 안 바뀐 이유

실습에서 `:visited`의 색(하늘색)이 처음엔 적용되지 않는 것처럼 보였다. 알고 보니 **브라우저에 그 링크의 방문 기록이 있어야** `:visited`가 적용된다. 해당 사이트를 여러 번 방문해 기록이 남은 뒤에야 하늘색으로 바뀌는 걸 확인할 수 있었다.

- 즉 `:visited`는 "내가 이 링크를 눌러본 적 있는가"라는 브라우저 히스토리에 의존한다.
- 새 시크릿 창이나 기록을 지운 상태에서는 계속 `:link`(빨강)로 보인다.

> 순서 팁: 보통 `:link → :visited → :hover → :active` 순으로 적는다. (앞 글자만 따서 **L-V-H-A**, "LoVe HAte"로 외우기도 한다.)

## 오늘의 한 줄 정리

링크는 상태가 바뀌는 요소라, `:hover`로 상호작용 느낌을 주고 `:visited`로 방문 여부를 표시하면 사용자 경험이 좋아진다. 단, `:visited`는 실제 방문 기록이 있어야 보인다.

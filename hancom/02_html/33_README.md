

# 클로드 코드(Claude Code) 설치 및 사용기

## 1. 구독 준비

클로드 코드를 사용하려면 아래 중 하나가 필요하다.

- **Pro 플랜** (월 $20, 연간 결제 시 월 $17)
- **Max 플랜** (5x / 20x)
- **Team Premium** 시트 (Team Standard는 클로드 코드 미포함)
- **Enterprise** 플랜
- 또는 API 종량제(Console PAYG) 결제

즉 Free 플랜과 Team Standard 플랜에는 클로드 코드가 포함되지 않으므로, 최소 Pro 이상 구독이 필요하다.

> 참고: 시스템에 `ANTHROPIC_API_KEY` 환경 변수가 설정되어 있으면 구독이 아닌 API 과금으로 인증될 수 있으니, 구독 사용량으로 쓰고 싶다면 이 변수가 없는지 확인해두는 게 좋다.

## 2. 설치

공식 설치 가이드: https://code.claude.com/docs/ko/quickstart#winget

### Windows (winget)

1. cmd 창을 열어 `winget`이 설치되어 있는지 확인한다.
2. `winget install` 명령어로 클로드 코드를 설치한다.

```
winget install Anthropic.ClaudeCode
```


![](https://velog.velcdn.com/images/developjkj/post/08928ded-8ad7-453a-967c-7f6f2a84a7ce/image.png)


## 3. 첫 실행

cmd에서 `claude`를 실행하면 다음 순서로 진행된다.

1. **폴더 신뢰 여부 확인** — 현재 디렉토리를 신뢰할지 물어본다.
2. **계정 로그인/연동** — 웹 클로드(claude.ai)에서 사용하는 계정과 동일한 자격증명으로 로그인하면 Pro/Max 구독이 클로드 코드에 연결된다.
3. 이후 클로드 코드 세션이 시작된다.

![](https://velog.velcdn.com/images/developjkj/post/11a5321d-4659-4cd8-b689-c26fa60cb521/image.png)

세션이 시작되면 **Shift + Tab**으로 작업 모드를 전환할 수 있다.

- **Plan 모드**: 실제 코드를 작성하기 전에 설계(계획) 단계를 먼저 정리해준다. 사용자가 이 계획을 검토·수정한 뒤 승인하면, 그 계획을 바탕으로 클로드 코드가 실제 작업을 수행하는 방식이다.

## 4. 오늘 사용해본 명령어

| 명령어 | 설명 |
|---|---|
| `@` | 폴더 내부 특정 파일을 지정해 접근. 해당 파일을 대상으로 작업 요청 가능 |
| `/exit` | 현재 세션 종료 |
| `/resume` | 이전에 진행했던 세션 목록을 확인하고 이어서 열기 |
| `/clear` | 현재 세션(컨텍스트) 초기화 |

## 5. 추가로 알아두면 좋은 명령어 / 옵션

### 세션 관리

| 명령어 | 설명 |
|---|---|
| `claude -c` (`--continue`) | 현재 디렉토리의 가장 최근 세션을 바로 이어서 시작 |
| `claude -r` (`--resume`) | 세션 선택 화면을 열어 원하는 과거 세션을 골라서 이어감 |
| `claude -r <session-id>` | 특정 세션 ID를 지정해서 바로 이어감 |
| `/compact [지시사항]` | 대화 기록을 요약본으로 교체해 컨텍스트를 정리 (특정 부분에 집중해서 요약하도록 지시도 가능) |
| `/context` | 현재 컨텍스트(토큰)를 얼마나 사용 중인지 확인 |
| `/export` | 현재 대화를 클립보드에 복사하거나 텍스트 파일로 저장 |

### 기타 유용한 팁

- **Ctrl + W**: 세션 선택 화면에서 같은 레포의 모든 워크트리로 범위 확장
- **Ctrl + A**: 세션 선택 화면에서 머신 전체 프로젝트로 범위 확장
- 세션은 로컬의 `~/.claude/projects/` 아래에 프로젝트 디렉토리 단위로 저장되므로, 필요하면 직접 백업하거나 검색할 수도 있다.
- `CLAUDE.md` 파일을 프로젝트 루트에 만들어두면, 세션을 새로 시작하거나 이어갈 때마다 자동으로 불러와 프로젝트 규칙/컨텍스트를 매번 재설명할 필요가 없다.

## 정리

- 구독(Pro 이상) → 설치(winget) → 실행 → 계정 연동 → Plan 모드로 설계 후 작업, 이 흐름만 잡아두면 초기 세팅은 끝이다.
- `@`, `/exit`, `/resume`, `/clear`에 더해 `/compact`, `/context`, `/export`, `-c`/`-r` 정도만 추가로 익혀두면 실무에서 대부분의 상황을 커버할 수 있다.
